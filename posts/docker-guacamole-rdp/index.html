Win10 远程桌面 Html5 客户端 Guacamole
2020-07-13 18:49 folder undone   label tutorial   docker   guacamole   html5   rdp   security   hyper-v  

Win 10 远程桌面服务作为开放给外网的高权限系统组件存在很大的安全隐患，因此如果需要使用rdp服务，最好不要直接开放其对应的3389端口，而是通过Remote Desktop Gateway这个默认存在于rdp客户端的认证通道或者是通过代理来避免端口直接暴露。以上两种方式前者较麻烦，后者则造成每次连接远程桌面都要开启代理，都不太方便。本文使Guacamole将本地的rdp服务转换成html5内容，通过浏览器访问https内容来实现连接，不再需要服务端和向局域网开放3389端口。

本文搭建的环境和程序如下

EnvironmentContains
Win 10Hyper-v, IIS Webserver
Hyper-vVM Debian10 (Docker)
DockerGuacamole

No Need To Prepare Config File

More offcial Docs Here

This Part is for the native build therefore SHOULD BE SKIPPED. As we use docker file here, the default user has been set to guacadmin with password guacadmin.1

Install Guacamole by Docker

Docker Site and uploader’s Github

In Hyper-v (Debian10)

  1. Pull image from docker
docker pull oznu/guacamole
  1. Build a Dockerfile which is used for changing base url. Run docker build -t guapath . at the place where you put the Dockerfile to bulid a new image.
FROM oznu/guacamole:latest

RUN set -x \
&& rm -rf ${CATALINA_HOME}/webapps/ROOT \
&& mv ${CATALINA_HOME}/webapps/ROOT.war ${CATALINA_HOME}/webapps/rdp.war
  1. Run container (replace /root/guacamole with your config directory which is should mkdir first when new build) from your new-built image.
docker run -d \
  --name=win10rdp \
  -p 127.0.0.1:8080:8080 \
  -v /root/guacamole:/config \
  guapath

Nginx URL Rewrite

Offcial Document My original plan is to use IIS ARR3.0 module to achive the URL rewrite purpose as this IIS webserver also do the job of sharing my files in Lan via Webdav. But after a query online, I find out that the support of Websocket in IIS which Guacamole needed to communicate seems lack. Therefore I use Nginx to do reverse proxy both for Guacamole in Hyper-V and Webdav that host on IIS.2

In Hyper-v (Debian10)

Assign IPs to VM 3

To get double IPs in debian VM, first you need to add two network interfaces (here for example eth0 is external for Lan and the eth1 is internal to host) on hyper-v. Then login to the VM and add those lines to /etc/network/interfaces, eth1's static ip is different from the host 192.168.208.145 (like one number after) and mask is same as host’s (you can find those in System Networking Control Panel):

auto eth0
iface eth0 inet dhcp

auto eth1
iface eth1 inet static
    ip   192.168.208.146
    mask 255.255.255.240

Apply changes by systemctl restart networking. Type ip a in command to find your IPs.

Install Nginx with Webdav module

sudo apt-get install nginx-full -y

Remove Default Page

rm /etc/nginx/sites-enabled/default

Add Proxy Config

Add those lines to your /etc/nginx/nginx.conf file inside the server tag. Replace rdp with your guacamole's url path.

    location /rdp/ {
        proxy_redirect off;
        proxy_buffering off;
        proxy_http_version 1.1;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $http_connection;
        access_log off;
        proxy_pass http://127.0.0.1:8080/rdp/;
    }

Replace S4f_ with your IIS Webdav url path, and 192.168.208.145 with your Win 10’s internal ip. I also changed the query string from client to IIS to prevent problems4 by if set.

    # add in http
    map $http_destination $destination {
        ~*(https?:\/\/)([a-z0-9\.]+\/)(.+)$ http://192.168.208.145:8088/$3;
        default $http_destination;
    }
    # add in server
    location /S4f_/ {
        proxy_redirect      off;
        proxy_buffering     off;
        proxy_http_version  1.1;
        proxy_set_header    Destination          $destination;
        # don't set this header as webdav destination check needed
        # proxy_set_header    Host                 $http_host;
        proxy_set_header    X-Real-IP            $remote_addr;
        proxy_set_header    X-Forwarded-For      $proxy_add_x_forwarded_for;
        proxy_set_header    http_x_forwarded_for $remote_addr;
        proxy_set_header    remote_addr          $remote_addr;
        access_log          off;
        proxy_pass http://192.168.208.145:8088/S4f_/;
    }

Config HTTPS and Auth

Copy your server.crt and server.key to nginx/ folder (use scp /path/to/file username@a:/path/to/destination or Winscp). You can add extra auth for anyone who visit the site by nginx’s basic auth.5 Replace server config with following lines.

server {
    listen       443 ssl http2;
    server_name  localhost;
    server_tokens off;
    ssl_certificate     server.crt;
    ssl_certificate_key server.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;
    ssl_ciphers         HIGH:!aNULL:!MD5;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
}

And to disable default nginx index page, add in server:

    location / {
        return 404;
    }

As a summary, your nginx.conf file should look like this:

# Default settings and others
http {
    # Default but remember to delete SSL-related settings
    # Because it's included in server
    map $http_destination $destination {
        # ..
    }
    server {
        # ...
        location / {
            return 404;
        }
        location /rdp/ {
            # ...
        }
        location /S4f_/ {
            # ...
        }
    }
}

When done, restart your Nginx service:

sudo systemctl restart nginx

Manage Nginx and Guacamole to Autostart

For Nginx:

# nginx autostart (enable)/ disable (disable)
sudo systemctl enable nginx
# nginx start/stop/status
sudo systemctl start nginx

To check port use lsof -i -P.The Nginx log is in /var/log/nginx/ as access.log and error.log for debug.

For Guacamole, in /etc/systemd/system/docker-win10rdp.service add:

[Unit]
Description=Docker for win10rdp
Requires=docker.service
After=docker.service

[Service]
Restart=always
ExecStart=/usr/bin/docker start -a win10rdp
ExecStop=/usr/bin/docker stop -t 2 win10rdp

[Install]
WantedBy=default.target

Enable the service:

sudo systemctl enable docker-win10rdp.service

Edit Guacamole Settings

In Hyper-v (Debian10)

Add Account

Access Guacamole's web page using the ip address https://192.168.51.24/rdp with guacadmin for both username and password.
After a successfully login, open settings and click on the Users. Create your own account with all permissions checked and delete the default guacadmin account.

Add Connections

Click on the New Connection inside the Connections. Fill the Name, Protocol, Hostname (Better not to use ip), Port, Username, Password, Security Mode and check the Ignore server certificate. Save the changes.

Test on Connection

Back to Home, click on the connection you just added. Check if everything is fine.

  4. Description: related discussion and webdav path requirement

    In Nginx’s reverse proxy part S4f_, I add extra filter to change query uri for Webdav service. I’m using rclone as my Webdav client, however, it contains bugs while sending requests to a proxyed server. For example, rclone sends method MOVE and query uri https://<nginx>/S4f_/path/to/file to the server when rename a file while the query uri to rename without proxy is http://<IIS>/S4f_/path/to/file. Therefore, to fix the problem, I add extra if-statement and regex set to convert path that contains scheme and server name to relative path. Edit1: It seems that it’s a common problem when use Nginx as reverse proxy and not just the bugs of rclone. More information about this is in description above. Edit2: Because of RFC2518, Destination header is an absolute URI. Convert it to relative is not the solution. The better way is to change the scheme and path to IIS backend. Edit3: if-state in Nginx will double escape() the query string (like convert already converted %6E to %256E). So I use map (should place in http section) instead. For some reason, IIS Webdav server behind proxy will check if MOVE method’s query string Destination is match the proxy Host. To avoid error, delete or comment the proxy_set_header host $http_host in location /S4f_/ {..}. ↩︎
