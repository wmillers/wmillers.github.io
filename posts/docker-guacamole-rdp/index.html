Win10 远程桌面 Html5 客户端
2020-07-13 18:49 folder undone   label tutorial   docker   guacamole   html5   rdp   security   hyper-v  

Win 10 远程桌面服务作为开放给外网的高权限系统组件存在很大的安全隐患，因此如果需要使用rdp服务，最好不要直接开放其对应的3389端口，而是通过Remote Desktop Gateway这个默认存在于rdp客户端的认证通道或者是通过代理来避免端口直接暴露。以上两种方式前者较麻烦，后者则造成每次连接远程桌面都要开启代理，都不太方便。本文使用的工具将本地的rdp服务转换成html5内容，通过浏览器访问https内容来实现连接，不再需要客户端和向局域网开放3389端口。

本文搭建的环境和程序如下： |Environment|Contains| |-|-| |Win 10|Hyper-v, IIS Webserver| |Hyper-v|Docker(VM Debian10)| |Docker|Guacamole| 按外部访问流程排序 |Program|Usage|Environment|Address|Included| |-|-|-|-|-| |IIS Webserver|HTTPS auth and port redirect|Win 10|Lan:443/rdp|not included| |Guacamole|HTML5 RDP server|Hyper-v|172.17.111.38:8080|-| |RDP|Microsoft remote desktop|Hyper-v|127.0.0.1:3389|-|

Prepare config file

addtional document

First, prepare the config file that will be used by guacamole. Replace 172.17.111.33 with your Win 10 IP address connected to the Hyper-v (as 172.17.111.38 is the VM’s IP that Hyper-v assigned to). Also don’t forget to change the username and password (md5). In Debian, you can use md5sum <File contains PASSWORD> to hash your plain password. user-mapping.xml

<user-mapping>
    <!-- Per-user authentication and config information 
    <authorize username="USERNAME" password="PASSWORD">
        <protocol>vnc</protocol>
        <param name="hostname">localhost</param>
        <param name="port">5900</param>
        <param name="password">VNCPASS</param>
    </authorize>
    -->

    <!-- Another user, but using md5 to hash the password
         (example below uses the md5 hash of "PASSWORD") -->
    <authorize 
            username="admin"
            password="**PASSWORD CONVERT TO MD5**"
            encoding="md5">
    <connection name="Unique Name">
        <protocol>rdp</protocol>
        <param name="hostname">172.17.111.33</param>
        <param name="port">3389</param>
    </connection>
    </authorize>

</user-mapping>

Install Guacamole by Docker

Docker Site and uploader’s Github

  1. Pull image from docker
docker pull oznu/guacamole
  1. Run container (replace /root/user-mapping.xml with your config file)
docker run \
  -p 8080:8080 \
  -v /root/user-mapping.xml:/config \
  oznu/guacamole

Config URL rewrite

IIS Webserver to Guacamole

Install Application Request Routing (ARR) module

Install ARR 3.0 for Reverse Proxy support. You can install the ARR module directly from the location: http://www.microsoft.com/web/gallery/install.aspx?appid=ARRv3_0.

Change URL rewrite settings

  1. Open the IIS Manager and open the Default Web Site config pannel. Then double click on the URL Rewrite option.
  2. Add 2 rules for Inbound rules and another 2 for Outbound Rules by clicking on the Add Rule(s)... action and Choose the Blank Rule template. Contents that needed to be changed are in the following tables:

Inbound Rules |Name|Input|Match|Pattern|Action Type|Request URL|Stop Processing| |-|-|-|-|-|-|-| |S4f|-|match||-|-|-|

Create an Inbound Rule for the parent website (localhost in this example) with the following settings:

  • pattern: rdp(/)?(.*)
  • check the Ignore case checkbox
  • rewrite URL (VM’s IP) set to http://172.17.111.38:8080/{R:2}
  • check the Append query string checkbox
  • check the Stop processing of subsequent rules checkbox
